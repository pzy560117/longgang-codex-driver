# 验收示例

**功能**: FEAT-EXPORT-PLATFORM-001 统一导出平台
**最后更新**: 2026-05-13
**负责人**: ANALYSIS-001

> 本文件提供测试左移阶段的验收示例索引。具体自动化用例、契约文件和实现测试由后续 CONTRACT / implementation 任务落地。

## 核心场景

### AE-001 创建任务成功

- 关联需求: FR-001
- 正向示例: 给定认证上下文完整且 `taskCode` 已注册启用，当调用 `POST /api/export/tasks`，则 1 秒内返回全局唯一 `taskId`，任务为 `PENDING`，并记录创建审计。
- 负向示例: 给定 `taskCode` 未注册、已禁用、无权限或 `queryParams` 超过 32KB，当创建任务，则返回对应错误且不得落无效任务。
- 证据目标: 创建请求/响应、任务记录、幂等范围、审计日志。

### AE-002 查询任务进度

- 关联需求: FR-002
- 正向示例: 给定用户有任务可见权限，当查询任务详情，则返回状态、总数、已处理数、进度百分比和错误信息字段。
- 负向示例: 给定任务不存在或无权查看，当查询详情，则返回 404 或 403，不泄露其他用户任务信息。
- 证据目标: 详情响应、权限失败响应、进度快照。

### AE-003 下载文件

- 关联需求: FR-003
- 正向示例: 给定任务 `COMPLETED` 且文件未过期，当请求下载，则签名 URL 返回 `downloadUrl`、`expiresAt`、`fileName`、`fileSize`，stream 模式返回等价元信息。
- 负向示例: 给定文件过期、未发布、校验失败或无权限，当请求下载，则返回失效或拒绝结果，不返回下载地址。
- 证据目标: 文件元信息、签名响应、校验记录、下载审计。

### AE-004 查询历史任务

- 关联需求: FR-004
- 正向示例: 给定正式筛选维度，当普通用户查询历史，则只返回本人任务；管理员按权限查看全局任务。
- 负向示例: 给定测试层新增未确认筛选项，则必须标记为 assumption / gap，不得写成已验收口径。
- 证据目标: 分页响应、筛选参数、用户/管理员对比。

### AE-005 调度并发与 DB 抢锁

- 关联需求: FR-005
- 正向示例: 给定多个 worker 实例，当同一子系统待执行任务超过并发上限，则成功执行数不超过配置值。
- 负向示例: 给定多实例同时抢同一任务，只有一个实例获得有效锁。
- 证据目标: 抢锁记录、并发计数、数据库时间、审计。

### AE-006 分片、ZIP 与空数据

- 关联需求: FR-006
- 正向示例: 给定导出行数超过 20000，当执行导出，则自动分片并打包 ZIP；给定 0 行，则生成仅表头文件并完成。
- 负向示例: 给定渲染失败、字段校验失败或超过最大导出量，则任务失败或拒绝创建并记录错误码。
- 证据目标: 分片文件、ZIP 清单、空文件表头、阶段事件。

### AE-007 注册配置

- 关联需求: FR-007
- 正向示例: 给定管理员权限，当创建或更新导出注册配置，则 `taskCode` 唯一，启停、并发、保留期、阈值和格式策略可查询。
- 负向示例: 给定无权限或禁用配置，当操作或创建任务，则返回 403 或 `TASK_DISABLED`。
- 证据目标: registry 快照、配置摘要、启停响应。

### AE-008 集中查询模板

- 关联需求: FR-008
- 正向示例: 给定注册配置声明数据源、参数 schema、模板、字段映射、脱敏策略和游标字段，当执行导出，则平台按集中查询口径生成数据。
- 负向示例: 给定非法模板、未声明参数、不允许数据源或快照冲突，则任务失败并收口到产品定义错误码。
- 证据目标: 模板版本、参数摘要、字段映射摘要、批次检查点。

### AE-009 权限、数据范围和脱敏

- 关联需求: FR-009
- 正向示例: 给定认证上下文完整，当创建、查询、下载或执行查询，则平台按 `operatorId`、`tenantId`、`roleCodes`、`orgScope` 施加权限和数据范围。
- 负向示例: 给定权限不足、认证字段缺失或脱敏失败，则拒绝操作或任务失败，并记录审计。
- 证据目标: 403 响应、dataScopeExpression、脱敏校验证据。

### AE-010 审计与事件追踪

- 关联需求: FR-010
- 正向示例: 给定任务完整执行，当查看审计和事件日志，则 CREATE、DISPATCH、EXECUTE、DOWNLOAD、CANCEL、RETRY 等动作可按 `requestId` 串联。
- 负向示例: 给定失败任务，必须记录错误码、失败阶段和上一成功阶段。
- 证据目标: 审计日志、任务事件日志、failure triage。

### AE-011 过期清理

- 关联需求: FR-011
- 正向示例: 给定文件超过保留期，当 cleanup job 执行，则先标记不可下载再删除对象。
- 负向示例: 给定对象删除失败，则保留可重试记录且下载返回失效结果。
- 证据目标: 失效标记、清理日志、重试记录。

### AE-012 取消和重试

- 关联需求: FR-012
- 正向示例: 给定 PENDING 任务，当取消，则最终状态为 `CANCELED`；给定 FAILED 任务，当重试，则沿用原 `taskId` 并递增 `attemptNo`。
- 负向示例: 给定 PENDING/EXECUTING 任务收到重试请求，则返回 `INVALID_TASK_STATE` 且不得重复派发执行。
- 证据目标: 状态机记录、attemptNo、审计链路。

### AE-013 执行尝试、锁租约和配置快照

- 关联需求: FR-013
- 正向示例: 给定任务创建后配置发生变化，当执行或失败重试，则沿用创建时配置快照，并记录 `attemptNo`、`lockOwner`、`lockExpireAt`。
- 负向示例: 给定锁租约过期，接管实例必须延续当前 `attemptNo` 并从检查点继续。
- 证据目标: configSnapshotDigest、锁租约、批次检查点。

### AE-014 幂等冲突

- 关联需求: FR-013
- 正向示例: 给定相同幂等范围和相同参数摘要，当重复创建任务，则返回原 `taskId`。
- 负向示例: 给定相同幂等范围但参数摘要不同，则返回 `IDEMPOTENCY_CONFLICT`，不得创建新任务。
- 证据目标: idempotencyScope、requestDigest、冲突响应。

### AE-015 锁续租和接管

- 关联需求: FR-005 / FR-013
- 正向示例: 给定 worker 抢锁成功，当批次执行中，则在批次边界续租并记录 `leaseRenewedAt`。
- 负向示例: 给定租约未过期，其他实例不得抢占；租约过期后只能一个实例接管。
- 证据目标: 数据库时间、锁字段、接管审计。

### AE-016 配置变更隔离

- 关联需求: FR-007 / FR-013
- 正向示例: 给定注册配置更新，当创建新任务，则新任务使用新配置；旧任务和失败重试继续使用原快照。
- 负向示例: 给定快照与模板或字段映射冲突，不得静默放宽字段、格式或阈值。
- 证据目标: 新旧配置快照、冲突证据。

### AE-017 采购订单样板注册

- 关联需求: FR-014
- 正向示例: 给定 `purchase-order-export` / `purchase` 注册启用，当提交创建时间范围、订单状态、供应商、采购组织和关键词条件，则按样板查询合同创建导出任务。
- 负向示例: 给定缺失游标字段、字段顺序错误或敏感字段策略缺失，则不得进入成功交付。
- 证据目标: 样板 registry、查询模板、字段定义、脱敏策略。

### AE-018 采购订单边界和压测

- 关联需求: FR-014
- 正向示例: 给定 `0/1/20000/20001/100000` 行测试数据，则分别验证空表头、单文件、分片 ZIP 和 10 万行压测记录。
- 负向示例: 给定 `100001` 行且配置未放开，则拒绝创建或进入更严格控制流。
- 证据目标: 边界数据、总耗时、批大小、分片数、锁续租次数、文件校验值。

### AE-019 查询执行上下文串联

- 关联需求: FR-008 / FR-010 / FR-013
- 正向示例: 给定查询批次执行，当查看日志，则任务快照、尝试上下文、模板版本、数据源编码和批次检查点可串联。
- 负向示例: 给定批次重试耗尽，则错误收口并保留重试次数和退避时间。
- 证据目标: 查询日志、审计日志、批次检查点。

### AE-020 执行阶段事件

- 关联需求: FR-006 / FR-010 / FR-013
- 正向示例: 给定任务成功执行，则事件包含 `QUERY_READY`、`QUERY_BATCH_DONE`、`FILE_PART_WRITTEN`、`PACKAGE_DONE`、`FILE_VERIFIED`、`DELIVERY_READY`。
- 负向示例: 给定任一阶段失败，则记录失败阶段、错误码和上一成功阶段。
- 证据目标: 阶段事件序列、失败阶段证据。

### AE-021 文件发布和校验

- 关联需求: FR-003 / FR-006 / FR-011
- 正向示例: 给定文件渲染完成，当校验通过，则从临时对象发布为可下载对象并记录校验时间。
- 负向示例: 给定元信息缺失、校验值不匹配或发布失败，则不生成下载地址并进入失败处置。
- 证据目标: tempStorageKey、publishedStorageKey、checksum、publishedAt、deliveryReadyAt。
