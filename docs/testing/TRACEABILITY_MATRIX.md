## 规则

- 每条 P0/P1 需求都必须恰好出现一次，或被明确拆分。
- 每一行都必须指向至少一个测试层级和一个证据路径。
- API / 数据行必须包含契约或 schema 校验证据。
- 当前仓库无业务代码时，测试层级允许标注为 `contract`、`planned` 或 `blocked-by-contract`，但不得伪装成已有 `src` 实现。
