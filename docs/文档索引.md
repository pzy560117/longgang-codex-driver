# 文档索引

本文用于快速说明 `docs/` 下各目录和关键文件的用途。新增文档时优先使用中文短文件名；已有英文文件暂不强制重命名，先在本索引中给出中文说明。

## 目录说明

| 目录 | 说明 | 优先读者 |
| --- | --- | --- |
| `docs/operations/` | 生产部署、生产接入、手工启动、迁移和交接材料。 | 接手研发、运维、DBA |
| `docs/testing/` | 验证矩阵、测试策略、测试数据、验收报告和证据边界。 | 测试、研发、评审 |
| `docs/product/` | PRD、验收标准、需求追溯、状态矩阵和难点研究。 | 产品、研发、评审 |
| `docs/architecture/` | 架构约束包和生产实现边界。 | 架构、研发、评审 |
| `docs/context/` | 仓库地图、架构 brief、feature pack 等上下文入口。 | 新接手研发 |
| `docs/harness/` | Driver、task、trace、规则治理和知识集成协议。 | Harness 维护者 |
| `docs/knowledge/` | 可复用经验、决策、规范和风险记录。 | 全体研发 |
| `docs/reviews/` | 阶段 review 结果和 findings。 | 评审、修复负责人 |
| `docs/aicoding-workflow-v1/` | AI Coding 工作流对外沟通材料。 | 团队讨论、汇报 |

## 后续命名建议

- 新增中文文档优先放中文文件名，例如 `生产接入交接指南.md`。
- 已被 `task.json`、trace、测试或脚本引用的英文文件，先不要零散重命名；需要重命名时单独开任务统一更新引用。
- 每次新增面向人读的文档，都同步更新本索引。
