# AI 生图提示词模板

## 标准工作流

AI 生图不能直接替代前端规格。UI-facing feature 必须按以下顺序执行：

1. 完成 `docs/product/requirement-interface-matrix.md`，确保每条 P0/P1 需求都映射到页面、状态、字段、操作、接口和验收。
2. 完成 `docs/product/difficulty-research.md`，提前研究高风险 UI、数据、交互、性能和权限难点。
3. 完成 `docs/design/design-brief.md`，写清 CSS / token 要求：颜色、字体、字号、间距、圆角、阴影、状态色、信息密度和响应式。
4. 使用 `docs/design/ai-image-brief.md` 生成或编辑多状态界面图。
5. 使用 `docs/design/ui-image-review.md` 评审图片。未通过时继续编辑图片，不进入代码。
6. 使用 `docs/design/image-to-frontend-spec.md` 把通过的图片转换为前端规格，再交给 Codex 实现代码。

## 默认语言与文字质量

除非用户明确指定其他语言，AI 生图提示词必须默认要求**简体中文界面文案**。

写入 `docs/design/ai-image-brief.md` 时必须包含以下约束：

- 所有可见 UI 文案、标题、按钮、标签、表格列名、状态、错误提示默认使用简体中文。
- 英文只允许用于必要品牌名、产品名、专有名词、技术缩写或真实 API/代码字段；业务界面文案不要中英混排。
- 禁止乱码、伪中文、无意义拉丁字母、拼音占位、Lorem ipsum、随机字符、错别字密集的小字。
- 对容易生成乱码的小字号长文本，改成短中文词组或结构化数据，不要求图像模型渲染大段正文。
- 提示词必须给出关键中文文案清单，让图像模型照抄，而不是自由编造。

`docs/design/ui-image-review.md` 必须检查文字质量：只要出现明显乱码、假字、伪中文、错误中英混排或不可读关键文案，图片不得进入代码阶段，必须重新编辑图片提示词并再生成。

## 基础提示词结构

```
生成"[页面名称]"的 UI 设计图

页面类型：[列表页/详情页/表单页/仪表盘/登录页]
设计风格：[Ant Design/Element Plus/Material Design]
设备类型：[桌面端/移动端]（宽度）
输入真相源：[PRD / requirement-interface-matrix / design-brief / design-tokens / component-map / screen-states]

语言与文字质量：
- 默认使用简体中文生成所有可见 UI 文案
- 必须照抄关键中文文案清单，不要自由编造小字
- 不要出现乱码、伪中文、拼音占位、Lorem ipsum 或随机英文字母
- 小字号说明文字必须短句化，避免大段不可读文字

必须生成以下状态的设计图：
1. 正常态（有数据）
2. 空态（无数据）
3. 加载态
4. [其他需要的状态]

页面元素：
- 顶部区域：[描述]
- 内容区域：[描述]
- 底部区域：[描述]

边界场景：
- 文本超长：[处理方式]
- 数据超多：[处理方式]

CSS / Token 要求：
- 颜色：[来自 design-tokens]
- 字体/字号：[来自 design-brief]
- 间距/圆角/阴影：[来自 design-tokens]
- 表格/表单密度：[来自 design-brief]

交互说明：
- [关键交互]

禁止：
- 不要添加需求矩阵中没有的功能
- 不要省略 required state
- 不要使用与 Design Tokens 冲突的样式
- 不要生成乱码、伪中文、随机字符或不可读关键文案
```

## 图片编辑提示词结构

```
基于上一版 UI 图继续编辑，不改变已通过评审的信息架构和组件结构。

需要修复：
- [来自 ui-image-review.md 的失败项]
- 如有乱码/伪中文/不可读关键文案，逐项替换为明确中文短文案

保持不变：
- [已通过的布局、组件、颜色、状态]

补充状态：
- [需要补充的页面状态]

禁止：
- 不要新增 Product Spec 没有的功能
- 不要改变已通过评审的需求映射
- 不要保留任何乱码、伪中文、拼音占位或随机字符
```

## 列表页示例

```
生成"订单列表页"的 UI 设计图

页面类型：列表页
设计风格：Ant Design 5.x 风格，简洁专业
设备类型：桌面端（1440px宽度）

必须生成以下状态的设计图：
1. 正常态：显示10条订单数据
2. 空态：无订单时的引导界面
3. 加载态：骨架屏效果

页面元素：
- 顶部：面包屑、页面标题、"新建订单"按钮
- 筛选区：状态下拉、时间范围、搜索框
- 表格列：订单号、客户名、金额、状态、时间、操作
- 底部：分页器

边界场景：
- 订单号超长：最多20字符+省略号
- 客户名超长：最多10字+省略号

状态标签颜色：
- 草稿：灰色
- 待审核：橙色
- 已通过：绿色
- 已拒绝：红色
```

## 表单页示例

```
生成"新建订单"表单页的 UI 设计图

页面类型：表单页
设计风格：Ant Design 5.x 风格
设备类型：桌面端（1440px宽度）

必须生成以下状态的设计图：
1. 初始态：空表单，显示必填项标记
2. 校验错误态：显示字段错误提示
3. 提交中态：提交按钮显示loading

页面元素：
- 顶部：面包屑、页面标题
- 表单区域：基本信息卡片、订单信息卡片
- 底部操作栏：取消、保存草稿、提交

表单规则：
- 必填字段标记红色星号
- 错误提示显示在字段下方
```

## 移动端示例

```
生成"订单列表"移动端页面的 UI 设计图

页面类型：列表页
设计风格：iOS 风格
设备类型：移动端（375px宽度）

必须生成以下状态的设计图：

交互说明：

- 上拉加载更多
- 左滑显示删除
```
