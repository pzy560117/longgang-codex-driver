# 青岛城市学院模板正文要求提炼

本文件只提炼正文内容要求，不处理字体、行距、页边距、页码等版式问题。

## 摘要

1. 中文摘要一般 `300-600` 字。
2. 以第三人称撰写。
3. 必须包含“目的、方法、结果、结论”四要素，重点放在结果和结论。
4. 建议按三段写：
   - 研究目的与意义
   - 主要研究内容、研究方法
   - 结果和结论
5. 中文关键词 `3-5` 个。
6. 英文摘要应与中文摘要内容一致。
7. 摘要从行业痛点转向系统方案时必须有过渡解释，避免直接从“摄影师经营痛点”跳到“三类角色系统”。
8. 摘要中技术栈只保留核心技术，不展开工具清单。
9. 摘要末段应写系统达到的研究结果和结论，不写“当前版本”“本次测试”等过程记录。

## 目录与章节骨架

模板给出的正文骨架为：

1. 第1章 绪论
2. 第2章 关键技术介绍
3. 第3章 系统分析
4. 第4章 系统设计
5. 第5章 系统实现
6. 第6章 系统测试
7. 结论
8. 参考文献

## 第1章 绪论

应包含：

1. 研究背景
2. 研究意义
3. 国内外研究现状
   - 1.3.1 国外研究现状
   - 1.3.2 国内研究现状
4. 研究内容
5. 论文组织结构

要求：

1. 绪论约占正文 `10%`
2. 一般不超过 `1000` 字
3. 研究现状要列举 `2-3` 个代表性成果，并做引文标注
4. 研究内容和论文组织结构应拆成两个小节，不建议合并为“研究内容与论文结构”
5. 正文编号可采用 `1.`、`(1)`、`①` 的层级形式，但同一篇论文内必须保持一致
6. 引用编号、西文作者名、中文与英文之间的空格样式需要全文统一

## 第2章 关键技术介绍

内容精简，主要介绍开发使用的技术和工具。

要求：

1. 约占正文 `5%`
2. 不要长篇展开

## 第3章 系统分析

应包含：

1. 可行性分析
   - 技术可行性
   - 经济可行性
   - 社会可行性
   - 可按需要补充其他可行性
2. 功能性需求分析
3. 非功能性需求分析

推荐目录结构：

```text
3.1 可行性分析
3.1.1 技术可行性
3.1.2 经济可行性
3.1.3 社会可行性
3.1.4 其他可行性
3.2 功能性需求分析
3.3 非功能性需求分析
```

硬性要求：

1. 功能需求分析中**必须画用例图**
2. 可按角色或功能模块拆分小节
3. 本章约占正文 `20%`

## 第4章 系统设计

应包含：

1. 系统架构设计
2. 系统功能结构设计
3. 系统主要功能模块设计
4. 数据库设计
   - E-R 模型
   - 表结构设计

推荐目录结构：

```text
4.1 系统架构设计
4.2 系统功能结构设计
4.3 系统主要功能模块设计
4.4 数据库设计
4.4.1 系统 E-R 模型（或概念结构）
4.4.2 表结构设计
```

硬性要求：

1. 必须有系统架构图
2. 必须有功能结构图
3. 主要功能模块设计至少介绍 `6` 个核心功能模块
4. 必须有 E-R 图
5. 核心数据表至少介绍 `6` 个
6. 数据表必须与 E-R 图一致
7. 本章约占正文 `25%-30%`

## 第5章 系统实现

可按用户功能模块、管理员功能模块等方式展开。
若项目本身为典型前后端分离系统，也可在本章内部按“前端实现”和“后端实现”展开，再在小节中细分核心模块。

硬性要求：

1. 至少介绍 `6` 个核心功能模块
2. 每个模块要同时写：
   - 功能描述
   - 实现方法
3. 前端实现部分要有必要截图，且截图应来自**浏览器实际运行界面**
4. 后端实现部分可附少量核心代码片段或代码图
5. 不能只写功能现象，必须解释实现过程
6. 页面截图不能只是罗列页面外观，必须说明页面完成了什么业务动作
7. 关键代码不能大段堆砌，必须围绕一个实现点进行解释
8. 本章约占正文 `25%-30%`
9. 同一章内系统实现截图大小应尽量一致

## 第6章 系统测试

应包含：

1. 测试方法
2. 测试环境
3. 功能测试
4. 性能测试
5. 测试结果

硬性要求：

1. 功能测试要提供测试用例表，包含输入、预期输出、实际输出
2. 功能测试应聚焦核心功能模块
3. 性能测试可包含压力测试、安全测试等
4. 测试结果要分析发现的问题及解决方案
5. 本章约占正文 `10%-15%`

## 结论

应包含：

1. 主要成果总结
2. 创新点或特色总结
3. 不足分析
4. 改进方向与未来展望

约束：

1. 结论应精炼、完整、准确
2. 不得夸大结论
3. 若无法得出明确结论，可写讨论与分析

## 参考文献

1. 执行 GB/T 7714-2015
2. 至少 `10` 篇
3. 只列正文中真实引用过的文献
4. 应包含一定比例的近五年文献
