# 最终检查清单

## 1. 内容与体裁

- [ ] 摘要是否还出现命令、日志、类名、接口路径、组件名
- [ ] 第一章是否仍然像开题报告复述，而不是论文绪论
- [ ] 第二章是否只写了与本文直接相关的技术
- [ ] 第三章是否存在接口清单、类名堆砌、需求文档腔
- [ ] 第四章是否围绕关键问题组织，而不是围绕源码目录
- [ ] 结论是否避免了“首次”“领先”“填补空白”等无据夸大表述

## 2. 图表与代码清单

- [ ] `mermaid` 是否都已经导出为正式图片
- [ ] 图题是否在图下，表题是否在表上
- [ ] 图号、表号、代码清单编号是否连续
- [ ] 文中出现的“如图/表 x-x 所示”是否真的有对应对象
- [ ] 图文件是否都有正文引用和解释段
- [ ] 页面截图是否做了裁剪或重点标注，而不是原样堆放
- [ ] 若存在“系统实现”章，前端截图是否来自浏览器真实运行界面，而不是原型图或静态拼图
- [ ] 代码清单是否只保留少量核心片段，而非大段贴代码
- [ ] 若存在“系统实现”章，后端代码片段是否都围绕关键实现点展开并配有说明

## 3. 引文与参考文献

- [ ] 正文是否在研究背景、研究现状、关键技术、设计或测试中真实使用了引文
- [ ] 正文引文编号是否按首次出现顺序连续编号
- [ ] 同一文献重复引用时是否沿用了原编号，而不是重新编号
- [ ] 参考文献是否全部真实可检索
- [ ] 文中引用过的文献是否都已著录
- [ ] 未引用文献是否已移除
- [ ] 文末参考文献列表是否按正文首次引用顺序排列
- [ ] 正文编号与文末列表编号是否一一对应，无跳号、重号或顺序错位
- [ ] 参考文献数量是否满足模板最低要求
- [ ] 若模板未明确更高要求，参考文献是否至少达到 15 条
- [ ] 中文参考文献占比是否约为 60%
- [ ] 英文参考文献占比是否约为 20%
- [ ] 其他参考文献占比是否约为 20%
- [ ] 若按 15 条基线执行，是否至少达到“中文 9 条、英文 3 条、其他约 3 条”
- [ ] 官方文档、RFC、技术手册等是否只作为补充来源，而没有占据主体
- [ ] 是否包含一定比例的近五年文献
- [ ] GB/T 7714-2015 格式是否统一

## 4. 测试与结果

- [ ] 测试章节是否包含“测试方法、测试环境、功能测试、性能测试、测试结果”
- [ ] 功能测试是否提供了输入、预期输出、实际输出
- [ ] 功能测试是否聚焦核心功能和边界场景
- [ ] 性能测试若未真实执行，是否已如实说明
- [ ] 测试结果是否写明发现的问题与处理方式
- [ ] 构建通过是否被正确写成“构建校验”而非“测试结果”

## 5. 模板与专项一致性

- [ ] 模板规定的章节骨架是否被保留
- [ ] 用例图、系统架构图、功能结构图、E-R 图、核心表结构是否满足模板要求
- [ ] 开题设想与实际实现差异是否已被显式交代
- [ ] 未落地功能是否没有被写成已实现事实
- [ ] 致谢中是否混入模板说明或操作说明
- [ ] 中英文摘要内容是否一致
