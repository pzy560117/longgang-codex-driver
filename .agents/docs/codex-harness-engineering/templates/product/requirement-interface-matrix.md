# Requirement Interface Matrix 模板

> 目标: 确保需求分析完成后，每一条需求都能追溯到具体界面、界面状态、字段、操作、接口和验收证据。没有映射完成前，不进入设计或实现阶段。

## 5. 通过标准

- [ ] 每个 P0/P1 需求至少映射到一个页面、状态或接口。
- [ ] 每个页面、弹窗、抽屉、嵌入视图都有需求来源。
- [ ] 每个 required state 都有触发条件、界面表现和验收标准。
- [ ] 没有“界面有功能但需求没有”的多余实现。
- [ ] 所有缺口已关闭，或明确标记 BLOCKED。
