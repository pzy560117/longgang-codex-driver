# Global AGENTS.md

## 通用工作原则

- 默认使用简体中文输出。
- 在没有验证证据前，不要声称任务已完成。
- 先阅读代码和文档，再修改文件。
- 开始实现前，先确认验收标准、测试入口和完成证据路径。
- 尽量保持改动范围小且可审计。

## 工具与命令规则

- 先用 `Get-Command` 检查命令是否存在。
- 命令不存在时优先使用已知绝对路径，不盲目重装工具。
- 搜索文件优先使用 `rg` 或 `rg --files`。
- 不要用内联脚本绕过正常文件和脚本结构。

## Git 规则

- dirty workspace 默认视为阻塞，除非用户明确说明哪些改动属于当前任务。
- 禁止在未确认时使用破坏性命令，如 `git reset --hard`、`git checkout --`、递归删除。
- 提交前必须有 fresh verification evidence。

## 安全规则

- 不把 token、cookie、私钥、密码写入仓库、trace 或文档。
- 不在不可信环境使用高权限 sandbox。
- 对外部输入保持最小权限原则。

## 分层规则

- 跨项目稳定习惯放全局 `~/.codex/AGENTS.md`。
- 当前仓库规则放项目根 `AGENTS.md`。
- 长文档放项目 `docs/`，不要把所有细节塞进入口文件。

## 不应放在全局的内容

- `task.json`、`progress.txt`、`codex-loop.ps1` 的项目 workflow。
- 某个仓库专属的提交规则、目录结构、接口名称。
- 某个产品专属业务规则。
