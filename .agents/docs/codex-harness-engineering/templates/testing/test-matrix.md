## 覆盖规则

- 每条 P0/P1 需求都必须出现在这个矩阵中。
- 每条 P0/P1 行都必须指向一个验收示例或场景 ID。
- 每个必需 UI 状态都必须映射到 story、组件、E2E、视觉一致性，或给出明确豁免。
- API / 数据变更必须映射到契约和 API 集成证据。
- 计划中的实现任务应能够从这个矩阵推导出 affected tests。
- `planned` 或 `blocked` 的 P0/P1 行在最终 Verify 前必须给出解释。
