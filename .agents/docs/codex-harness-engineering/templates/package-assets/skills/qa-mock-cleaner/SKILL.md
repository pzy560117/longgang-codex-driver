# QA Mock Cleaner 前后端连通器

## 核心职责
在进行严格的端到端浏览器测试之前，必须保证 "Zero-Mock"（零假数据拦截）。本 Skill 的唯一作用就是根据 QA 提供的清洗清单，安全、无损地拆除各个组件中的 Mock，恢复原本的网络层轴心。

## 执行前置条件
当前特性开发的前端页面中存在虚拟后端支持，并且 `e2e-plan.md` 第一章 (Mock 依赖排雷清单) 已经生成。

## 工作流 (Workflow)

1. **获取排雷计划**
   - 读取目标路径中的 `e2e-plan.md` 文件的 "Mock 依赖排雷清单" 章节。
   - 精准定位被指出的那些 `.vue` 页面文件或 `src/mock/` 目录下的拦截文件。

2. **精细化剥离与真实接口替换**
   - 针对文件内的 `<MockStateWrapper>` 进行剥离，改为通过 axios 或 Vue store 调用真实后端接口的错误/加载捕获机制。
   - 如果遇到全局拦截（如 MSW 或 axios-mock-adapter），执行完全关闭操作，保证接下来的流量必须直通后端环境。

3. **联通自检**
   - 确认关键文件的修改未破坏 Vue 组件编译 (`npm run build` 或依赖自身验证手段不报错)。

4. **完成与交接**
   - 在完成剥离动作后，通知用户清理工作结束，并转交权柄给 `qa-e2e-runner` 去启动测试运行。
