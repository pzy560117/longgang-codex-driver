# Ticket Filter Demo PRD-Lite

## 1. 背景

- 业务背景: 客服主管需要在工单列表中快速筛选高优先级工单，并保存常用视图。
- 当前问题: 每次进入系统都要重复配置筛选条件，效率低且容易遗漏。
- 为什么现在做: 当前团队的工单量增长，手动筛选成本过高。

## 2. 目标

- 业务目标: 提升主管查看和分派工单的效率。
- 用户目标: 可以按状态、优先级、负责人筛选工单，并保存自己的常用视图。
- 不做什么: 不做跨用户共享视图；不做复杂图表分析。

## 3. 用户与角色

| 角色 | 使用场景 | 权限差异 |
| --- | --- | --- |
| 主管 | 筛选、保存视图、批量分派 | 可创建和删除自己的视图 |
| 客服 | 查看工单列表 | 只能使用系统默认筛选 |

## 4. 范围

- In scope: 列表筛选、保存视图、空态/错误态、无权限限制
- Out of scope: 共享视图、导出 Excel、复杂报表

## 5. 页面 / 流程影响

- 受影响页面: 工单列表页
- 受影响流程: 工单查询、保存筛选视图
- 受影响接口: `/tickets`、`/ticket-views`

## 6. 成功标准

- 主管可以保存视图并刷新后继续使用
- 筛选结果可正确展示空态、加载态和错误态

## 7. 风险与依赖

- 主要风险: 设计稿只覆盖 happy path 会导致状态缺失
- 外部依赖: 需要统一 OpenAPI contract 和 generated client
